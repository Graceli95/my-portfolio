# Portfolio Website - User Stories

## Epic 1: Navigation & Core Structure

### US-1.1: Global Navigation
**As a** visitor  
**I want to** see a clear navigation menu on every page  
**So that** I can easily explore different sections of the portfolio

**Acceptance Criteria:**
- Navigation visible on all pages
- Links to: Home, About, Projects, FAQ, Events, Contact
- Active page highlighted
- Mobile hamburger menu for screens < 768px
- Smooth scroll to sections when applicable

---

### US-1.2: Footer Information
**As a** recruiter  
**I want to** see Grace's contact information and social links in the footer  
**So that** I can quickly connect via my preferred platform

**Acceptance Criteria:**
- Footer present on all pages
- LinkedIn, GitHub, and Email links working
- Copyright notice with current year
- Professional tagline displayed
- Quick navigation links included

---

## Epic 7: Contact Page

### US-7.1: Contact Form
**As a** recruiter or potential collaborator  
**I want to** send Grace a message directly from her website  
**So that** I can reach out without opening my email client

**Acceptance Criteria:**
- Form fields: Name (required), Email (required, validated), Message (required)
- Submit button
- Email validation (proper format required)
- All fields required before submission
- Clear error messages for invalid inputs
- Success message after successful submission: "Thank you! I'll get back to you soon."
- Error message if submission fails
- Form clears after successful submission
- Responsive design

---

### US-7.2: Form Backend Integration
**As a** site owner (Grace)  
**I want to** receive emails when someone submits the contact form  
**So that** I can respond to inquiries promptly

**Acceptance Criteria:**
- Form submits via Next.js API route
- Email sent using Resend or SendGrid
- Email includes: sender name, sender email, message content
- Email delivered to Grace's inbox
- Error handling if email service fails

---

## Epic 8: Performance & Technical

### US-8.1: Fast Page Load
**As a** visitor on any connection speed  
**I want to** pages to load quickly  
**So that** I have a smooth browsing experience

**Acceptance Criteria:**
- Page load time < 3 seconds on 4G
- Images optimized (WebP format)
- Lazy loading implemented for images
- Code splitting for large components
- Lighthouse performance score > 85

---

### US-8.2: Mobile Responsiveness
**As a** mobile user  
**I want to** have a great experience on my phone  
**So that** I can browse the portfolio anywhere

**Acceptance Criteria:**
- Works on screens 320px and up
- Touch targets at least 44x44px
- Text readable without zooming
- No horizontal scrolling
- Mobile navigation (hamburger menu)
- All features functional on mobile

---

### US-8.3: Accessibility
**As a** user with accessibility needs  
**I want to** navigate the site using keyboard and screen readers  
**So that** I can access all content

**Acceptance Criteria:**
- WCAG 2.1 Level AA compliance
- All interactive elements keyboard accessible
- Proper heading hierarchy (H1-H6)
- Alt text for all images
- Color contrast ratio â‰¥ 4.5:1
- Form labels properly associated
- Focus indicators visible
- Lighthouse accessibility score > 90

---

### US-8.4: SEO Optimization
**As a** site owner (Grace)  
**I want to** my portfolio to rank well in search engines  
**So that** recruiters can find me when searching for candidates

**Acceptance Criteria:**
- Each page has unique title tag
- Each page has meta description
- Proper heading hierarchy on all pages
- Sitemap.xml generated
- Robots.txt configured
- Open Graph tags for social sharing
- Semantic HTML used throughout

---
